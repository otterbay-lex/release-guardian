# CODE_REVIEW.md

## Readability

- 初级开发者能否理解规则、结论和修复步骤。
- 名称是否清晰，函数和章节是否职责单一。
- 主 `SKILL.md` 是否保持精简，详细规则是否按需拆分。

## Structure

- 项目识别、规则执行、证据收集、评分和报告是否分离。
- 工程质量是否与合规检查同等受到重视。
- 通用规则和项目专项规则是否能组合而不重复。

## Maintainability

- 新增项目类型或检查项时是否无需重写核心流程。
- 风险标识、严重程度和报告字段是否稳定。
- 脚本是否避免不必要依赖并有失败回退路径。

## Risk

- 是否可能打印完整密钥或个人信息。
- 是否把关键词命中错误认定为真实风险。
- 是否把“未发现”错误表述为“已合规”。
- 是否未经确认修改文件、安装依赖或执行发布。
- 是否检查了与实际发布物不同的对象。

## Scope Control

- 不加入未经请求的自动部署、付费服务或外部账号操作。
- 不因合规深化而弱化构建、测试、依赖和可恢复性检查。
- 不进行无关重构，不引入未经批准的依赖。

## Verification

- 运行格式和结构检查。
- 使用静态网站、Node 项目、AI 应用、Codex Skill、假阳性和制品不一致样本测试。
- 对比原版与新版，确认新版提高覆盖率且没有明显增加误报。
- 人工检查报告是否适合小白理解，且证据和检查边界完整。

## Design Compliance

每次评审都要对照已批准设计的成功标准和后续实施计划，确认没有遗漏工程、发布、安全、合规、知识产权或运维维度。
