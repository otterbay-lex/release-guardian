# Release Guardian

面向初级开发者和非合规专业人士的 AI 发布质量守门员，在代码、网页、App、API、Skill 或其他制品公开前，主动检查工程质量、发布完整性、安全、合规、知识产权与可恢复性。

## 目标用户

- 希望把项目发布到 GitHub 或部署上线的初级开发者。
- 不熟悉安全、隐私、开源许可证和发布工程的产品负责人。
- 需要一套通用发布前复核流程的 AI Agent 使用者。

## 当前阶段

当前处于设计与项目初始化阶段。本阶段包括项目规则、架构边界和完整产品设计；暂不包含新版 Skill、扫描脚本或自动发布功能。

## 核心流程

1. AI 识别项目类型、发布目标和实际制品。
2. 默认执行快速体检，正式发布前执行深度检查。
3. AI 主动完成可执行的检查并收集证据。
4. 输出工程与合规并重的分级报告。
5. 用户确认修复批次后，由 AI 修改并重新验证。
6. 真正的推送、部署或发布作为独立授权动作执行。

## 如何使用

当前没有可执行程序。设计文件位于：

`docs/superpowers/specs/2026-07-15-release-guardian-v2-design.md`

## 如何验证当前阶段

- 确认根目录五份项目启动文件齐全。
- 阅读设计文档，确认没有占位内容或未决产品要求。
- 后续实施应先制定文件级计划，再分阶段开发和测试。

## 重要边界

- 检查通过不代表绝对安全或完全合规。
- Skill 不代替专业安全审计或正式法律意见。
- 未经用户确认，不修改项目、不安装依赖、不推送或部署。
- 能由 AI 完成的只读检查应由 AI 主动完成，尽量减少用户操作。
