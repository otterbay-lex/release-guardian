# 评分与报告规范

## 六维评分

- 工程质量：25
- 发布完整性：20
- 安全与供应链：20
- 隐私与合规：15
- 知识产权：10
- 运维与可恢复性：10

分数帮助排序，不能覆盖硬阻断。未检查的维度标记“需要人工确认”，不得默认为满分。

## 允许的结论

- 可以发布：无阻断，关键检查有证据。
- 有条件发布：无明确阻断，但有重要警告或待确认事项。
- 暂不建议发布：有重大工程、安全、合规或恢复缺口。
- 禁止自动发布：存在密钥、真实数据、高危漏洞等硬阻断。

## 发现明细

每项重要发现使用：

```text
[严重程度] [rule_id]：[标题]
状态：已验证 / 发现问题 / 需要人工确认 / 不适用
证据：[file:line or command]
可信度：高 / 中 / 低
为什么重要：[plain Chinese]
可能后果：[realistic consequence]
修复方式：[steps]
AI 可执行：[全部 / 部分 / 否]
用户动作：[only what AI cannot do]
复核方式：[exact check]
```

只列最重要的详细证据，不重复打印秘密或个人信息。修复后报告已解决项和剩余项，并重新计算结论。
