# 安全与供应链规则

## 凭证与私密配置

扫描已跟踪文件、待发布文件、环境文件、日志、示例和制品。覆盖通用 API_KEY/SECRET/TOKEN/PASSWORD/PRIVATE_KEY，以及 OpenAI、Anthropic、Google、AWS、Azure、GitHub、Stripe、Supabase、Firebase、腾讯云、阿里云等常见格式。

关键词只是线索。结合长度、格式、上下文和占位标记判断。真实或高度疑似真实的凭证为阻断项；报告不得输出完整值。若可能进入 Git 历史，建议轮换并检查历史，不要只删除当前文件。

## 应用安全

按项目类型检查认证、授权、最小权限、输入验证、注入、XSS、CSRF、路径遍历、公开管理接口、默认密码、错误堆栈、CORS/CSP 和上传处理。没有足够证据时标为需要人工确认，不宣称安全。

## 供应链

- 检查 manifest 与锁文件、依赖来源、Git URL、安装脚本和包发布内容。
- 可用现有工具时检查已知漏洞；联网或安装新工具需要确认。
- 区分“数据库未查到”“没有运行漏洞扫描”和“已验证无已知漏洞”。
- 确认的恶意包、来源劫持或当前可利用的严重漏洞为阻断项。
