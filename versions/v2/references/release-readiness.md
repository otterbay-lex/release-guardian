# 发布完整性规则

## 文档与版本

检查 README、安装与卸载、运行示例、配置说明、`.env.example`、版本号、CHANGELOG/Release Notes、许可证入口、支持范围、已知限制和反馈渠道。

版本必须来自实际 manifest 或发布元数据。不要仅因版本为 `0.x` 就判定过期；结合变更记录和发布意图判断。

## 制品一致性

明确区分源码、构建目录、压缩包、容器镜像和商店安装包。检查最终制品是否夹带：

- `.env`、数据库、缓存、日志、调试符号或测试账号。
- 不需要的源码、内部文档、私有证书或本机配置。
- 与已检查源码不同的版本或构建时间。

最终制品无法取得或与源码不一致时，标为阻断，不得用源码扫描结果替代制品结论。

## 升级与回滚

涉及数据库、持久数据、自动更新或配置格式变化时，检查迁移、兼容、备份和回滚。可能破坏用户数据却没有恢复路径时为阻断项。
