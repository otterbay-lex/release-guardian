# 隐私与中国合规初步规则

## 适用边界

本模块提供面向中国大陆发布的初步风险提示，不构成正式法律意见。发现高风险或事实不清时，说明需要法务、隐私或安全专业人员确认。

## 数据处理

识别项目是否收集、存储、传输或公开：姓名、联系方式、账号标识、设备信息、日志、位置、身份/金融/健康信息、未成年人信息、聊天内容、上传文件和生物识别信息。

检查：

- 处理目的是否明确且与功能必要性相符。
- 告知、同意、撤回、查询、更正和删除机制是否与实际处理一致。
- 敏感个人信息是否有单独必要性和更严格保护。
- 日志、分析、崩溃报告和客服材料是否脱敏。
- Cookie、第三方 SDK、模型服务和数据共享是否披露。
- 数据保存期限、删除、出境和境外服务商是否需要进一步评估。
- 面向未成年人或中国公众的服务是否涉及额外义务。

## 结论纪律

字段名不等于真实个人信息；示例数据需结合上下文。无法从代码确认隐私政策、备案、许可或平台后台设置时标记“需要人工确认”。明知需要授权或许可却完全未处理，且将直接公开运营时，可以构成阻断项。
